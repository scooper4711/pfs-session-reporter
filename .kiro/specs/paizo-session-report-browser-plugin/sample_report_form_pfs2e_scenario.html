Instructions

1. Type in event name (if there isn't one, typically there will be).

2. As noted above, each session is forced numerically by the order in which you report it and isn't necessarily the order in which the sessions were played.

3. Select the scenario of the first session you are reporting.

4. Input the date on which the scenario was played, the GM's Pathfinder Society ID #, and the GM's name (which should auto-populate from the GM's Pathfinder Society ID # if that GM is registered in our system). If the GM is taking credit for that session as part of his GM Rewards, then add the GM's character number and that character's faction.

5. Input the first character's Pathfinder Society ID # plus appropriate dash (-1, -2, etc) for 1st, 2nd, etc character.

6. The character's name should auto-populate if you typed the ID # correctly and the character currently exists in our system (it's possible that it may not).

7. The character's faction should auto-populate if you typed the ID # correctly and the character currently exists in our system (it's possible that it may not).

a. If the character you're reporting does not auto-populate, be sure to enter that character's name and faction correctly.

8. Input the number of prestige that character received.

9. If that character died, mark them dead (this will gray out other entries for that character).

10. Repeat steps 5 through 9 for each additional character that played in that session.

11. If there are more characters at the table than space available in the session report, click "Add Extra Character."

12. Click "Save Session," "Save and goto new session," "Save and Exit," "Cancel Changes," or "Cancel and Exit."

a. "Save Session" saves your data inputted so far and keeps you on that session's reporting screen.
b. "Save and goto new session" saves the current session and moves to the next highest numbered session automatically.
c. "Save and Exit" saves your progress and exits the reporting system entirely.
d. "Cancel Changes" resets the session you are currently reporting but keeps you on the same session.
e. "Cancel and Exit" cancels all changes and exits the reporting system entirely.

13. If a character or GM is marked as being disputed, that means that the reported player or game master has denied playing in or running this event. Please make sure that the reported data accurately reflects the player or GM that attended this session (not merely that it matches the session sheet) before confirming the report. If you are unsure who actually attended, please leave it disputed.
